Tejuino.css

A light-weight CSS framework that lets you jump right into the action

Tejuino is a framework that allows the user a wide range of configuration in their page's HTML without the need of writing too complex css. Tejuino provides a logical approach when assigning the page's layout without sacrificing any possible displays. With a mobile-first approach, Tejuino aims to provide an homogeneous experience across multiple devices.
Resets
Tejuino contains a dedicated reset file that provides a blank HTML look to the page; this means that almost all HTML elements have been reset to a basic property, i.e.: no margins & no padding. If there are elements that the framework does not modify in a direct way, their reset properties will still apply.
General Display Properties
Tejuino's main elements are 3: the container, the row and the column.
Tejuino approaches web development as if every element in a web page is either one of those three. Containers carry rows, columns or both, rows separate the different possible paragraphs inside the containers, and columns provide a general element view for the page.

Containers
Containers are the bulk of Tejuino; providing a clear way of separating elements in a page, containers hold together rows, columns and even other containers. They have a padding of 10px in all directions, and a margin of 0 to top & bottom and auto to the sides.
Rows
Rows provide a better way of differentiating elements inside the containers; more specifically, and as its class suggest, rows. A container can have as many rows as the user considers, and they have a 10px padding. They will also always start at the left of the element and will align its content to the center.
More Config
Rows can also provide its content with an specific direction with three self explanatory classes:
  • row-left
  • row-center
  • row-right
Columns
Columns are probably the most important element out of the main three. They let the user set a general view, order the elements inside a row and provide the option to set collapse markers for mobile view.
Columns divide its parent container into 12 different sections, and can be set to occupy all or part of that space.

Collapsing Cols
As mentioned earlier, cols can be configured to collapse within set markers. They allow an easy transition between mobile and desktop versions of the page. Currently, Tejuino supports four different markers:
  • col-sm will collapse with a minimum width of 558px
  • col-md will collapse with a minimum width of 768px
  • col-lg will collapse with a minimum width of 992px
  • col-xl will collapse with a minimum width of 1200px
Try adjusting the page to see how they work.

Set Attributes
Tejuino allows the user to set some specific properties to the target HTML element to provide it with specific behaviours, some of them are:
set-flex
This class is self explanatory; it'll provide the element with a display: flex style.
set-hidden
This class will provide the element with a visibility: hidden style. Best used with JS.
set-none
This class will attach to the element a display: none style, but for the user to implement it, it has to specify its size. The size markers are the same as the ones at the columns, with the same breakpoints:
  • set-sm-none triggers when the page's min width is 558px
  • set-md-none triggers when the page's min width is 768px
  • set-lg-none triggers when the page's min width is 992px
  • set-xl-none triggers when the page's min width is 1200px
set-full
This class works in conjunction with either a container or a row, and it'll their padding to 0.
Texts
Tejuino uses 10px as a base measurement for everything related to the page, from fonts to spacing. It's recommended that users stick to the rem as the default measure tool for additional css files.
Although it's recommended that the user use div as the default tag, p tags are also supported, providing a margin: 0.75 0
Defaults
Tejuino uses Trebuchet MS as its default font, with Lucida both Sans Unicode, Grande and Sans (in that order) as a fallback. Its also configured to have a font-size of 1.6rem (16px), a font-height of 1.6, and a font-weight of 400, with 0 margin & padding.
Images
Images in Tejuino are configured to provide a 100% width in relation of its parent container, with a height that auto adjusts automatic.
Hyperlinks
Hyperlinks are not that different from the regular HTML tag, with a few exceptions. They'll inherit their parent's colour and it will change to blue #004c9c when hovered; they'll also provide no text-decoration when hovered: like this!
Titles
Titles provide a way to differentiate and highlight parts of the page; they come in 6 sizes, from h1 to h6. All titles provide a margin of 0.5rem both up and down, and 0 to the sides, a line-height of 1.8, and a width of 100%. For each six h tag, the font-size decreases:

h1: 5rem or 50px

h2: 4.2rem or 42px

h3: 3.6rem or 36px

h4: 3rem or 30px

h5: 2.4rem or 24px
h6: 1.8rem or 18px
Inputs
Inputs are stylized depending on their type, but they also contain a general configuration: they'll inherit both their color & font-size from their parent element, and will provide a margin & padding of 0.25rem (2.5px) up and down and 0.5rem (5px) to the sides; they'll also occupy 80% of the width of their parent element. All inputs won't show their outline when focused.
Text Inputs
Text inputs include the types text, password, email and date, including its variants; these inputs have a #eaeaea solid border of 0.1rem (1px) with 0.5rem (5px) of border-radius.
Button Inputs
Button inputs include the types button, reset and submit; button inputs are configured to have their default background-color of #dddddd, but they also contain a padding of 1.5rem (15px) 0. Additionally, they also have their border removed and a border-radius of 2rem or 20px. They'll change the cursor and their background-color to #aaaaaa when hovered or active.

File Inputs
File inputs just include the file type; file type inputs are notoriously hard to stylize in general, to overcome this, a label element its necessary to accompany it. In Tejuino, file inputs are hidden, and their functionality its passed to their respective label; that label its stylized similar to a button: it has the same default background-color and its alternative when the label its active; it has a border-radius of 0.75rem or 7.5px and it'll center the text inside of it.

Special Inputs
Special inputs include the color, checkbox, radio and range types; there's currently any support for these kinds of inputs, so they'll behave like their regular HTML tags with the general input configuration.
Labels
Labels for inputs will only modify the font-weight of the text, modifying it to 700.
Buttons
Buttons provide a way to separate sections in a page, or simply to decorate parts of it; because of that, they've been separated into two main categories: the normal buttons that were designed around page decoration and the link buttons that were design around page navigation.
All buttons have a border-radius of 1rem (10px), have a padding of 1rem (10px) 2rem (20px), and provide a font-size of 1.5rem or 15px. They'll also occupy 80% of their parent container's width, and won't have any border.
